# public-skills

Claude Code / Codex CLI 向けの公開スキルライブラリ。

## プロジェクト概要

- `skills/` にスキルを格納。各スキルは `skills/<name>/SKILL.md` 形式
- [Agent Skills](https://agentskills.io) オープンスタンダード準拠
- Claude Code (`.claude/skills/`) と Codex CLI (`.agents/skills/`) の両方で利用可能

## ディレクトリ構成

- `skills/` — 公開スキル本体
- `docs/` — 方針・ガイドライン
- `.claude/` — このリポジトリ管理用ハーネス（Claude Code）
- `.codex/` — このリポジトリ管理用ハーネス（Codex CLI）

## スキル設計原則

- SKILL.md フロントマター標準に準拠（name, description, allowed-tools 等）
- 1スキル1ディレクトリ。補助ファイル（テンプレート、スクリプト）は同ディレクトリ内
- description は250文字以内。Claude が自動適用を判断できる具体的キーワードを含める
- スキル本文は500行以下。詳細は別ファイルに分離
- Mac/Windows両対応。シェルスクリプトはPOSIX互換またはクロスプラットフォーム配慮

## 基本思想

以下の思想に基づき行動する。

### 段階的開示
- 必要なときに必要なだけ情報を見せる
- SKILL.md は簡潔に。詳細は補助ファイルに分離
- README はスキル一覧の概要のみ。詳細は各SKILL.md参照

### 透過性
- 各スキルの想定用途・制約・既知の問題を明記する
- 判断根拠や却下理由をコメント・PRに残す

### AIの優れた提案を優先
- 指示追従より良いアイデアの提示を重視
- 優れた代替案があればpros/cons付きで提案する

### 多様性は善
- 同じ目的に複数のアプローチがある場合、各案のメリット・デメリットを明示して選択肢を残す
- 特定のツール・手法に偏らない

### 実践先行
- 完璧な設計より動くスキルを先に
- 実際に使って改善するサイクルを回す

### 自己評価
- スキル追加・変更後に品質を自己評価する。省略厳禁

## スキル品質基準

- frontmatter の name, description は必須
- description がスキルの用途を正確に表現しているか
- 副作用のあるスキルは `disable-model-invocation: true` を検討
- セキュリティリスクのあるコマンドを含まないか
- クロスプラットフォーム互換性があるか

## 開発プロセス

- main への直接push は軽微な修正のみ。新スキル追加は feature ブランチ → PR
- PR には追加スキルの概要と想定ユースケースを記載
